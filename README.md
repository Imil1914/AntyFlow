# AntyFlow

Локальный AI-холст: бесконечная доска (как Figma / Miro), где ноды — это заметки,
документы, чаты с ИИ, канбан-доски, генерация изображений, презентации,
Jupyter-ноутбуки, PDF с Q&A и встроенные AI-агенты. Стек: **Electron + React + tldraw**.

Всё работает локально на вашей машине; модели подключаются по вашим API-ключам
или локально через **LM Studio / Ollama** (LLM) и **ComfyUI** (картинки).

---

## Возможности

- **Заметки и документы** с рендером Markdown; режим Obsidian-хранилища (реальные `.md`).
- **ИИ-чат-ноды** с любой OpenAI-совместимой моделью (свои ключи).
- **Канбан-доска** — колонки, карточки, перенос задач drag-and-drop.
- **Бэклог (мультиканбан)** — несколько досок в одной ноде + чат с моделью и кнопка
  «🤖 Сделать» на карточке (ИИ выполняет задачу).
- **Гибкие связи** между нодами (плавные bezier-кривые); контекст ноды передаётся
  по стрелке в подключённый ИИ-чат.
- **Генерация изображений** (через внешний ComfyUI — опционально).
- **Презентации** (слайды, экспорт PDF/PPTX) и **Mermaid-схемы**.
- **Jupyter/Colab-нода** — постоянный Python-kernel прямо в ноде.
- **PDF-нода** — аннотации, локальный RAG (Transformers.js) и Q&A по документу.
- **Голосовой ввод** — локальный Whisper (Transformers.js, офлайн).
- **Встроенные AI-агенты**: OpenCode, AnythingLLM, OpenScience.
- **Синхронизация досок между устройствами** (через папку-облако) и
  **real-time совместная работа** (через свой sync-сервер на Cloudflare — см. ниже).

## Требования

- **Node.js 18+** (обязательно — сборка и работа).
- **Git** (нужен агенту AnythingLLM — клонируется автоматически).
- **Windows 10/11** (сборка настроена под Windows/NSIS).
- *Опционально:* **Python 3** (для Jupyter-ноды), **LM Studio** (локальные LLM),
  **ComfyUI** (локальная генерация картинок).

## Установка и запуск

```bash
git clone <URL-репозитория>
cd Flow
npm install
```

**Режим разработки** (hot-reload):
```bash
npm run dev
```

**Собрать установщик** (`.exe` под Windows → папка `release/`):
```bash
npm run dist
```
> Важно: перед `npm run dist` **закройте запущенный `Flow.exe`** — иначе
> electron-builder не сможет перезаписать `release/win-unpacked` (ошибка `EBUSY`).
> Просто `npm run build` собирает веб-часть в `out/`, но **не** пересобирает `.exe`.

## Первый запуск: API-ключи

1. Командное меню **Ctrl+K** → «Настройки провайдеров».
2. Включите провайдера, впишите **baseURL** и **свой API-ключ**. Любой
   OpenAI-совместимый эндпоинт:
   - OpenRouter — `https://openrouter.ai/api/v1`
   - LM Studio (локально, без ключа) — `http://127.0.0.1:1234/v1`
3. Ключи хранятся **локально** (`%APPDATA%/Flow/settings.json`), в репозиторий не попадают.

## Синхронизация между устройствами (без сервера)

Холсты можно синхронизировать через облачную папку:
1. В приложении → кнопка **«Файлы»** → выберите папку внутри
   OneDrive/Google Drive/Dropbox.
2. Туда пишутся заметки (`.md`) и снимки холстов (`.flow-canvas/*.json`).
3. На втором устройстве укажите **ту же** папку — всё подтянется.

Это не real-time (last-write-wins), но не требует сервера. Подробнее — файловый
слой в `src/main/canvas-sync.ts`.

## Real-time совместная работа (sync-сервер)

Несколько человек работают с одной доской одновременно (курсоры, живые правки).
Сервер — папка **[`sync-server/`](sync-server/)** (Cloudflare Worker на `@tldraw/sync`,
SQLite Durable Object — **работает на бесплатном плане Cloudflare**, R2 не нужен).

**Развернуть свой сервер:**
```bash
cd sync-server
npm install
npx wrangler login                 # откроется браузер — войдите в свой Cloudflare
npx wrangler deploy                # выведет адрес вида https://flow-sync.ВАШ.workers.dev
```
> Перед первым `deploy` откройте в дашборде Cloudflare **Workers & Pages** — там
> один раз создаётся `workers.dev` субдомен (Account Details → Subdomain).

**Подключить приложение:** меню доски → **⚙ Сервер синхронизации** → впишите адрес
воркера как `wss://flow-sync.ВАШ.workers.dev`. Затем **🌐 Сделать общей** (получите
ID комнаты) / у второго участника **🔗 Подключиться** (тот же ID).

Полная инструкция — в [`sync-server/README.md`](sync-server/README.md).

> Что синхронизируется: всё содержимое нод (канбаны, бэклоги, заметки, история ИИ).
> Что **нет**: живые терминалы/kernel/webview и локальные модели — они у каждого свои.

## Встроенные AI-агенты

Агенты **не входят в репозиторий** — Flow ставит/подтягивает их при первом
использовании.

| Агент | Что делает | Как ставится |
|---|---|---|
| **AnythingLLM** | RAG-ассистент с базой документов | Flow сам клонирует и собирает при первом запуске (~несколько минут, ~1 ГБ). |
| **OpenScience** | Научный AI-воркбенч | `npm i -g @synsci/openscience`. |
| **OpenCode** | Агент для кода в реальном терминале | Установите `opencode` (см. его репозиторий). |

> **ComfyUI** и **LM Studio** не входят в проект. Flow к ним подключается по
> `127.0.0.1`, если вы их поднимете сами.

## Структура проекта

```
src/main/         — Electron main: IPC, провайдеры, агенты, notebook, vault, canvas-sync
src/preload/      — мост window.flow (contextBridge)
src/renderer/src/ — UI: App.tsx (холст/доски), shapes/FlowNodeShapeUtil.tsx (все ноды)
sync-server/      — Cloudflare Worker для real-time совместной работы
electron.vite.config.ts — конфиг сборки
```

## Где что хранится (не коммитится)

- `node_modules/`, `out/`, `release/`, `.wrangler/` — в `.gitignore`.
