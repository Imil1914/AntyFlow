# AntyFlow — что реализовано

Локальное десктоп‑приложение (Electron + React + tldraw): бесконечный AI‑холст с нодами,
оркестратором, RAG, веб‑моделями и таймлайн‑памятью проекта.

- `src/main/` — Electron main: IPC, провайдеры моделей, агенты, оркестратор, PDF, vault, AnythingLLM.
- `src/preload/index.ts` — мост `window.flow` (+ типы в `src/renderer/src/flow-api.d.ts`).
- `src/renderer/src/App.tsx` — холст, доски, сайдбар, таймлайн, диалоги.
- `src/renderer/src/shapes/FlowNodeShapeUtil.tsx` — все виды нод (один shape `flow-node`, поле `kind`).

---

## 1. Ноды холста

Единый shape `flow-node` с полем `kind`; данные — в `props.extra` (JSON). Виды нод:

- **Заметка / Документ** (`note`, `doc`) — Markdown.
- **ИИ‑ассистент** (`ai`) — чат с моделью по API, видит связанные стрелкой ноды как контекст.
- **Ответ** (`answer`) — карточка ответа ИИ, «паутина» ответов.
- **Список** (`list`/`listcard`), **Канбан** (`kanban`), **Бэклог** (`board`), **Таблица** (`sheet`,
  с формулами `=SUM/AVG/…`), **Схема** (`diagram`, Mermaid), **Код** (`code`/`codeblock`),
  **Поиск** (`search`), **Картинка/Референс** (`image`/`ref`), **Презентация** (`deck`),
  **Jupyter‑ноутбук** (`notebook`, .ipynb, запуск кода).
- **Агент‑ноды**: **OpenCode** (терминал), **OpenScience**, **AnythingLLM** (webview‑воркбенчи).
- **PDF** (`pdf`) — просмотр + локальный RAG по документу (эмбеддинги, Q&A по выделению).
- **Оркестратор** (`orchestrator`) и его узлы (`orchtask`, `orchcall`).

Контекст по стрелке: любая нода, соединённая стрелкой с ИИ/оркестратором, попадает к нему в контекст
(`extractNodeContext`).

---

## 2. Провайдеры моделей по API

- OpenAI‑совместимые провайдеры: **LM Studio**, **OpenRouter**, **Z.ai GLM**, **Cherry** и др.
- Ключи/настройки в `%APPDATA%/flow/providers.json`, `settings.json`.
- Единая точка вызова `callModel` (main) — переиспользуется нодами и оркестратором.
- Формат модели `providerId::modelId`; vision (картинки‑референсы), ретраи на 429/5xx.

---

## 3. Веб‑чат‑ноды с логином (ChatGPT / Gemini / GLM)

Группа сайдбара **«Веб‑чаты (логин)»**. Виды: `webgpt`, `webgemini`, `webglm`.

- Встроенный сайт в `<webview>` с **отдельной persist‑сессией** на провайдера
  (`persist:webllm-<p>`) — вход своим аккаунтом, логин сохраняется между запусками.
- **Видимость для API‑нод**: диалог веб‑чата снимается (innerText) в реестр `agentTranscripts`;
  соедини веб‑ноду стрелкой с обычной `ai`‑нодой — та увидит содержимое чата. Бейдж «📄 N»
  в шапке показывает объём снятого текста.
- **Ввод в ноде**: поле «➤» вписывает запрос прямо в веб‑чат и создаёт ноду‑ответ; «➕» —
  взять последний ответ отдельной нодой.
- **Мост оркестратора** `webLLMAsk` (contracts → worker → broker → IPC `orch:askWebLLM` /
  `orch:webLLMResult` → App драйвит нужную веб‑ноду через реестр `webLLMRegistry`).

---

## 4. Оркестратор (мульти‑агентный)

Брокер в main + воркеры (worker‑threads). Бюджет токенов, глубина рекурсии, human‑in‑the‑loop,
трейс/статусы — централизованно в main.

- **Ресерч‑фаза**: тема → грани → поиск статей (OpenAlex) → заливка в AnythingLLM → гипотезы.
  Статьи **не вываливаются нодами** на доску: грузятся в базу знаний AnythingLLM, на доску — нода
  базы знаний + сводка литературы (сводка идёт в контекст ТЗ).
- **Веб‑сборка** (`webbuild.ts`): при открытых веб‑чат‑нодах оркестратор **делегирует генерацию
  сильным веб‑моделям** (round‑robin по Gemini/GLM/ChatGPT), а при пустом ответе — фолбэк на
  локальную API‑модель. Строит **упорядоченный, пронумерованный набор артефактов**:
  обзор решений (таблица), что учитывать/избегать (список), требования (список),
  архитектура (Mermaid‑схема), **канбан по неделям**, **ТЗ по неделям** (документ),
  метрики (таблица), **ноутбук‑прототип с кодом и пояснениями**, презентация, ИИ‑ассистент, риски.
  Ноды создаются одной пачкой, кластерами по группам, справа от ресерча.
- **Лекция посекционно** (`lecture.ts`): план секций (JSON) → каждая секция пишется отдельным
  вызовом по микро‑структуре методики → сшивка (фолбэк — одиночный синтез).
- Режимы исполнения узлов: pipeline, actor‑critic, council, ensemble, recursive.

## 5. RAG и выбор проекта при загрузке файлов

- При **дропе текстовых файлов** (pdf/docx/pptx/txt/md/rtf) — диалог: класть ли в RAG и **в какой
  проект** (workspace AnythingLLM). Список воркспейсов подтягивается, можно создать новый;
  последний выбор запоминается.
- `ingestDocument` грузит файл с **корректным MIME** по расширению (не только PDF) в выбранный
  workspace (раньше был жёстко `Flow`).
- **PDF «только на доску»** помечается `noRag` и **не индексируется/не эмбеддится** локально.
- RAG‑БД файлов (AnythingLLM) — общая система, **может пересекаться между досками**.
- Локальный PDF‑RAG — отдельно, изоляция по одному `pdf_id`, хранение в `%APPDATA%/flow/pdf/`.

## 6. Таймлайн доски + память проекта

Меню доски → **«🗓 Таймлайн доски: вкл/выкл»** (флаг свой у каждой доски).

**Разметка (бесконечная координатная сетка, не ноды):**
- Ось Y = время: **1 день = полоса высотой 1700 px** (вмещает ~5 нод в высоту).
- **Начало отсчёта — день создания доски** (dayIndex 0); выше него ничего не рисуется (пусто).
- Дневные полосы — **полубесконечны вправо**, окрашены **по неделям** в пастельные тона.
- Слева от дневной зоны — **широкие колонки недельных и месячных окошек** (подписи), дальше —
  пустая свободная зона.
- Всё бесконечно вниз (будущее); подписи «день · неделя N»; сегодняшний день выделен.

**Память доски (глобальная, своя у каждой доски):**
- Хранение — `localStorage: flow-boardmem-<boardId>` (у другой доски своя память).
- Нода **«Память доски»** показывает накопленные выжимки; соедини стрелкой с ИИ/оркестратором —
  он получит всю память как контекст.
- **Выжимка дня** — ручная (панель снизу, действует на день в центре экрана) и **автоматическая**
  (прошедшие дни с контентом суммируются сами).
- **Авто‑сводки за неделю и за месяц** по их завершении (rollup дневных выжимок).
- Каждая новая выжимка учитывает прошлую память → «конец дня передаётся в следующий день»
  накопительно.

**Управление:**
- Панель снизу: дата · «Выжимка дня» · «очистить: день / нед / мес».
- **Удаление (сжатие) пустого дня/недели/месяца** — если период пуст, всё ниже сдвигается вверх
  (убирает пустое время).

## Хранение данных

- Настройки/ключи — `%APPDATA%/flow/` (`providers.json`, `settings.json`).
- Доски — локально; опционально real‑time через Cloudflare‑воркер (`sync-server/`).
- Память доски и флаги таймлайна — `localStorage` (по boardId).
- Локальный PDF‑RAG — файлы в `%APPDATA%/flow/pdf/`; AnythingLLM — `%APPDATA%/flow/anythingllm/`.

## Сборка и доставка

- `npm run dev` — разработка; `npm run build` — веб‑часть; `npm run dist` — установщик + `Flow.exe`.
- Обновление установленного приложения без переустановки — свап `app.asar` (+
  `app.asar.unpacked/out/main/orchestratorWorker.js` при правках воркера).
- Репозиторий: github.com/Imil1914/AntyFlow.
